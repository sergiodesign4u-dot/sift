Synthesis view

The app home and the tap-0 main-job surface: the ranked, trustable picture of what customers need most. It owns the canonical confidence indicator, theme card, and filter control. Source: ia/docs/pages/synthesis.md.

01

Blocks and states

#BlockPurpose
1App header (0.1)Synthesis active; primary destinations
2Filter / sort barsource, recency, confidence; sort by signal rank
3Ranked theme listtheme cards, each with the confidence indicator
4Select + Build briefthe call is one tap from the picture (to 6.1)

States (nodes)

  • Populated (default): the ranked list.
  • 2.1 First-run empty: one CTA, Connect a source (to 3.1). The activation fork (Flow 1).
  • 2.2 Loading / synthesizing: skeleton cards while ingest and cluster run.
  • 2.3 Error: synthesis failed; retry (to 2.2) or Sources (to 3.0). Never a dead-end.

03

Canonical components (owned here)

Defined once on this page, referenced elsewhere, never re-specified. Recorded in sitemap.md.

Confidence indicator (D2 mandatory)

Shows the evidence count n=X, plus a low signal badge when the evidence is thin. Day-one scope; source diversity and time range are v2. Not color-only: the count and badge word carry the meaning.

Referenced by 4.0, 4.1, 6.1, 6.4. Thin-evidence threshold [?] pending (a v2 tuning question).

Theme card

Label (in the customer's language), volume n=X, recency, the confidence indicator, and a drill affordance. The card links to 4.0; a select control adds it to a brief.

Referenced by 6.1 (selection), 6.4 (shared brief).

Filter / sort control

Filter by source, recency, confidence; sort by signal rank (default), volume, recency. A page-level control, not a facet system; an empty filtered result recovers by loosening, not a dead-end.

Reused on 3.0 Sources (lighter, status and type).

04

SEO and accessibility

  • noindex (private app), robots: noindex, follow, no schema.
  • The theme list is a semantic list; each card is a link to 4.0 with an accessible name.
  • The confidence indicator is not color-only (count text plus badge word), so it survives grayscale and colorblindness.
  • Filter controls are labeled; every target is at least 44px.
